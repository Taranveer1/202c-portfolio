Step by Step Guide to Building Your Own Portfolio Site as a Beginner

Create a stunning developer portfolio in minutes — absolutely no coding experience required.

Having a great portfolio is essential for developers of all skill levels. This comprehensive guide will walk you through creating a professional-looking developer portfolio website using a ready-made template and deploying it online — all for free and without writing a single line of code!

Whether you're a complete beginner or already have some projects to showcase, this guide will help you create a portfolio that impresses potential employers and clients.

What You'll Need

  • GitHub Account (free) - We'll use GitHub to store your website files and make updates
  • Vercel Account (free) - This will host your website so others can see it online
  • Basic Computer Skills - If you can browse websites and fill out forms, you already have all the skills needed!
  • 30-60 Minutes - Set aside some uninterrupted time to complete this guide

Note: You don't need any coding experience, design skills, or technical background. This guide is specifically designed for complete beginners!

1 Initial Setup

1.1. Create a GitHub Account

GitHub is a platform where developers store and share their code. We'll use it to host your portfolio files.

  1. Go to github.com/signup
  2. Enter your email address in the field provided
  3. Create a password (make sure it's secure but something you'll remember)
  4. Choose a username (this will be part of your portfolio URL, so choose wisely!)
  5. Complete the verification puzzle when prompted
  6. Click "Create account"
  7. Check your email and verify your account by clicking the link GitHub sends you

Tip: Your GitHub username will appear in your portfolio URL, so consider using your real name or a professional username that you wouldn't mind sharing with employers.

1.2. Use Our Portfolio Template

Instead of building a website from scratch, we've created a professional portfolio template that you can customize with your information.

  1. Go to github.com/dev-portfolio-template/minimal-portfolio
  2. Click the green "Use this template" button near the top right of the page
  3. Select "Create a new repository" from the dropdown
  4. For the repository name, enter: your-username.github.io
    • Replace "your-username" with your actual GitHub username
    • This specific format is important! It must be exactly your username followed by .github.io
  5. Make sure the "Public" option is selected (this is required for GitHub Pages to work)
  6. Click "Create repository from template"

Example: If your GitHub username is "jane-developer", you would name your repository "jane-developer.github.io"

What's happening behind the scenes: When you use our template, you're creating a copy of a professionally designed portfolio website that you can modify. The template includes a responsive design that looks great on all devices, accessibility features, and optimized loading times.

2 Customize Your Portfolio

2.1. Add Your Personal Information

Now comes the fun part! Let's personalize your portfolio with your own information.

  1. In your new repository, find and click on the file called config.json
  2. Click the pencil icon (Edit this file) in the top right corner of the file view
  3. Update the information with your personal details:
    • name: Your full name (e.g., "Jane Developer")
    • title: Your professional title or the role you're seeking (e.g., "Front-End Developer" or "Software Engineering Student")
    • about: Write 2-3 sentences about yourself, your skills, and what you're passionate about
    • location: Your city and country/state (e.g., "San Francisco, CA" or "London, UK")
    • email: Your professional email address
  4. Update your social media links:
    • For any platforms you use, replace the example URLs with links to your profiles
    • For platforms you don't use, change the value to an empty string like ""
    • Example: "twitter": "https://twitter.com/yourhandle" or "instagram": ""
  5. Customize the theme color if desired:
    • The themeColor field accepts hex color codes (e.g., "#3b82f6" for blue)
    • You can choose any color you like to personalize your site
    • For help finding hex codes, try HTML Color Codes
  6. When you're done making changes, scroll down to the bottom of the page
  7. Enter a short description of your changes in the "Commit changes" box (e.g., "Update personal information")
  8. Click the green "Commit changes" button

Formatting tip: Make sure to keep the quotes and colons in the file as they are. Only change the text between the quotes. For example, change "name": "John Doe" to "name": "Your Name".

2.2. Add Your Projects

Projects are the heart of your portfolio. Let's add some to showcase your skills.

  1. In your repository, find and click on the file called projects.json
  2. Click the pencil icon to edit the file
  3. The file contains sample projects that you'll replace with your own. Each project has the following structure: 
    {
  "title": "Project Title",
  "description": "A short description of what this project does",
  "tags": ["HTML", "CSS", "JavaScript"],
  "imageUrl": "images/projects/project1.jpg",
  "liveUrl": "https://example.com",
  "sourceUrl": "https://github.com/yourusername/project"
}
  4. For each project, update:
    • title: Give your project a clear, descriptive name
    • description: Explain what the project does and why you created it (2-3 sentences)
    • tags: List the technologies, languages, or frameworks you used
    • imageUrl: You can keep this as is for now (we'll update images later)
    • liveUrl: Link to the live version if available, or use "" if none exists
    • sourceUrl: Link to the GitHub repository if available, or use "" if none exists
  5. You can add additional projects by copying an existing project's format and pasting it as a new entry
  6. Make sure to separate each project with a comma, except for the last one
  7. Once you've updated the projects, scroll down and commit your changes

Don't have any projects yet? No problem! Here are some ideas for beginners:

  • Your portfolio website itself (the one you're creating now)
  • An online resume or CV you've created
  • A personal blog or website you've made
  • Coursework or tutorials you've completed
  • Small practice projects (even if they're simple)

Remember that having 2-3 quality projects is better than having many incomplete or low-quality ones.

2.3. Add Your Profile Picture

A professional photo helps make your portfolio more personal and memorable.

  1. Prepare a professional-looking square image of yourself
    • Recommended size: 500x500 pixels
    • Choose a photo with good lighting and a neutral background
    • Crop it to square dimensions before uploading
    • Save it as JPG or PNG format
  2. In your GitHub repository, navigate to the public/images folder
  3. Click on "Add file" then "Upload files"
  4. Drag and drop your image or click to select it from your computer
  5. Rename your image to profile.jpg (or keep the format consistent with the existing file)
  6. Enter a commit message like "Add profile picture"
  7. Click "Commit changes"

Profile picture tips:

  • Dress professionally or business casual
  • Use a recent photo that actually looks like you
  • Smile! A friendly expression makes you appear approachable
  • If you don't want to use your photo, a professional avatar or logo works too

3 Deploy Your Website

3.1. Deploy with GitHub Pages (Option 1)

GitHub Pages is the simplest way to make your portfolio visible online:

  1. In your GitHub repository, click on "Settings" in the top navigation
  2. In the left sidebar, click on "Pages"
  3. Under "Source", select "main" from the branch dropdown
  4. Keep the folder set to "/ (root)"
  5. Click "Save"
  6. Wait a few minutes for GitHub to build and deploy your site
  7. Once deployed, you'll see a message saying "Your site is published at https://your-username.github.io"

Important: It may take up to 10 minutes for your site to become available after you first publish it. If you don't see it right away, be patient and check again later.

3.2. Deploy with Vercel (Option 2 - More Features)

Vercel offers more advanced features and faster performance:

  1. Go to vercel.com/signup and sign up for a free account
  2. Choose "Continue with GitHub" to connect your GitHub account
  3. Authorize Vercel to access your GitHub account when prompted
  4. Once logged in, click the "Add New..." button and select "Project"
  5. Find and select your portfolio repository from the list
  6. Vercel will automatically detect the project settings, so you can leave all defaults
  7. Click "Deploy"
  8. Wait for the deployment to complete (usually takes less than a minute)
  9. When finished, Vercel will show you the URL where your site is live

Why choose Vercel? Vercel offers better performance, analytics, and custom domain support. It's also industry-standard for many professional developers.

3.3. Add a Custom Domain (Optional)

For a more professional look, you can connect your own domain name:

  1. Purchase a domain from a provider like Namecheap, Google Domains, or GoDaddy
  2. If using GitHub Pages:
    • In your repository settings, go to the "Pages" section
    • Under "Custom domain", enter your domain name
    • Click "Save"
    • Follow the instructions to set up DNS records with your domain provider
  3. If using Vercel:
    • Go to your project dashboard in Vercel
    • Click on "Settings" then "Domains"
    • Enter your domain name and click "Add"
    • Follow the instructions to set up DNS records or nameservers

Professional domain tips: For a personal portfolio, consider using your name (like janedeveloper.com) or a professional handle. This looks more professional than generic domains or subdomains.

3.4. Making Future Updates

Keeping your portfolio updated is important. Here's how to make changes:

  1. Go to your GitHub repository
  2. Navigate to the file you want to change (e.g., config.json, projects.json)
  3. Click the pencil icon to edit the file
  4. Make your changes
  5. Scroll down and commit your changes
  6. Your site will automatically update within a few minutes

Keep your portfolio current: Update your portfolio whenever you:

  • Complete a new project
  • Learn a new skill or technology
  • Change jobs or responsibilities
  • Receive certifications or complete courses
